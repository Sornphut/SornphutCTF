Lab1: SQL Injection

ช่องโหว่ที่เกิดจากการนำข้อมูลจากผู้ใช้ไปต่อกับคำสั่ง SQL โดยไม่ตรวจสอบความปลอดภัย

🎯 เป้าหมาย

  • เข้าใจหลักการของ SQL Injection
  • ทดลองโจมตีระบบ Login ที่ไม่ใช้ Prepared Statement
  • เรียนรู้วิธีป้องกันด้วย Parameterized Query

🧠 ขั้นตอนการทำงาน

  1. กรอก username และ password ในฟอร์ม
  2. ระบบจะ query ข้อมูลจากฐานข้อมูลโดยตรง
  3. หากใช้ payload เช่น ' OR '1'='1 จะ bypass การตรวจสอบ

🔒 Secure Version

ใช้ Django ORM หรือ Prepared Statement เพื่อป้องกันการต่อ string โดยตรง

ทดลองโจมตี (Vulnerable) ดูวิธีป้องกัน (Secure)
