Lab3: Cross-Site Request Forgery (CSRF)

ช่องโหว่ที่เกิดจากการที่ผู้ใช้ถูกหลอกให้ส่ง request โดยไม่รู้ตัว ทำให้ attacker สามารถทำ action แทนผู้ใช้ได้

🎯 เป้าหมาย

  • เข้าใจหลักการของ CSRF
  • ทดลองโจมตีด้วยการสร้างหน้า HTML ปลอมที่ auto-submit ฟอร์ม
  • เรียนรู้วิธีป้องกันด้วยการใช้ CSRF token

🧠 ขั้นตอนการทำงาน

  1. ผู้ใช้ login เข้าสู่ระบบ
  2. มีฟอร์มที่ไม่มี CSRF token เช่น <form method="post">
  3. Attacker สร้างหน้าเว็บปลอมที่ auto-submit ฟอร์มนี้ → ระบบทำงานโดยที่ผู้ใช้ไม่รู้ตัว

🔒 Secure Version

ใช้ {% csrf_token %} ใน Django template เพื่อป้องกันการโจมตี

ทดลองโจมตี (Vulnerable) ดูวิธีป้องกัน (Secure)
{% csrf_token %}
{% if message %} {% if "ถูกต้อง" in message %}
{{ message }}
{% else %}
{{ message }}
{% endif %} {% endif %}
