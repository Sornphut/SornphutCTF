Lab5: Insecure Direct Object Reference (IDOR)

ช่องโหว่ที่เกิดจากการเข้าถึงข้อมูลโดยตรงผ่าน parameter เช่น ?id=2 โดยไม่มีการตรวจสอบสิทธิ์ ทำให้ attacker เข้าถึงข้อมูลของผู้ใช้คนอื่นได้

🎯 เป้าหมาย

  • เข้าใจหลักการของ IDOR
  • ทดลองโจมตีด้วยการเปลี่ยนค่า parameter ใน URL
  • เรียนรู้วิธีป้องกันด้วยการตรวจสอบสิทธิ์ผู้ใช้ก่อนเข้าถึงข้อมูล

🧠 ขั้นตอนการทำงาน

  1. ผู้เล่น login เข้าสู่ระบบ
  2. เข้าหน้าโปรไฟล์ เช่น /lab5/?id=1
  3. เปลี่ยนค่า id เป็น 2 → ระบบ Vulnerable จะโชว์ข้อมูลของผู้ใช้คนอื่น
  4. ลองเปลี่ยนเป็น ?id=5 → ระบบจะโชว์ FLAG

🔒 Secure Version

ตรวจสอบ session และสิทธิ์ผู้ใช้ → ให้เข้าถึงเฉพาะข้อมูลของตัวเองเท่านั้น
