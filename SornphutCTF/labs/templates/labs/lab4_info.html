Lab4: File Upload Vulnerability

ช่องโหว่ที่เกิดจากการอนุญาตให้อัปโหลดไฟล์โดยไม่ตรวจสอบชนิดไฟล์ ทำให้ attacker สามารถอัปโหลดไฟล์อันตรายและรันโค้ดบนเซิร์ฟเวอร์ได้

🎯 เป้าหมาย

  • เข้าใจความเสี่ยงจากการอัปโหลดไฟล์โดยไม่ตรวจสอบ
  • ทดลองโจมตีด้วยการอัปโหลดไฟล์ .php หรือ .exe
  • เรียนรู้วิธีป้องกันด้วยการตรวจสอบ MIME type และจำกัดนามสกุลไฟล์

🧠 ขั้นตอนการทำงาน

  1. ผู้เล่นเลือกไฟล์จากเครื่องและอัปโหลดขึ้นเซิร์ฟเวอร์
  2. ระบบ Vulnerable จะบันทึกไฟล์โดยตรง → attacker สามารถรันโค้ดได้
  3. ระบบ Secure จะตรวจสอบนามสกุลไฟล์ เช่น .jpg, .png, .gif และเก็บไฟล์ในโฟลเดอร์ที่ไม่สามารถ execute ได้

🔒 Secure Version

ใช้การตรวจสอบชนิดไฟล์, จำกัดนามสกุล, และเพิ่ม CSRF token เพื่อป้องกันการโจมตี
