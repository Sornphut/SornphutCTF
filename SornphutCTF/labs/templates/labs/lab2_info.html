Lab2: Cross-Site Scripting (XSS)

ช่องโหว่ที่เกิดจากการนำ input ของผู้ใช้ไป render โดยไม่ sanitize ทำให้ attacker แทรก script ได้

🎯 เป้าหมาย

  • เข้าใจหลักการของ XSS
  • ทดลองโจมตีด้วยการ inject script ลงในฟอร์ม comment
  • เรียนรู้วิธีป้องกันด้วยการใช้ auto-escaping และ sanitization

🧠 ขั้นตอนการทำงาน

  1. กรอกข้อความในช่อง comment
  2. ระบบจะ render ข้อความนั้นบนหน้าเว็บ
  3. หากใส่ payload เช่น <script>alert('XSS')</script> จะทำให้ script ทำงาน
  4. กด View Page Source ดู HTML ที่ถูก render

🔒 Secure Version

ใช้ Django template auto-escaping หรือ library เช่น bleach เพื่อ sanitize input ก่อน render

ทดลองโจมตี (Vulnerable) ดูวิธีป้องกัน (Secure)
